### S0. 온보딩/로그인

- 목적: 동기화 기반 준비
- UI:
    - “로그인 없이 시작(로컬)” 옵션(개인 프로젝트라면 이거 넣으면 좋아)
    - Apple/Google 로그인
- 액션:
    - 로그인 / 게스트 시작
- 상태:
    - Error: 로그인 실패 → 재시도/다른 방법

---

### S1. 홈(피드)

- 목적: 저장된 클립을 **빠르게 훑고**, 다시 찾을 수 있게
- UI:
    - 상단: 검색바(탭 시 S6로)
    - 탭/드롭다운: “전체 / 폴더 선택”
    - 그리드(Masonry): 카드(대표 이미지, 텍스트 1줄, 도메인)
    - 플로팅 버튼(+): 수동 추가(있으면 좋음)
- 액션:
    - 카드 탭 → S4 상세
    - 폴더/태그 필터 → 리스트 갱신
- 상태:
    - Empty: “아직 저장된 클립 없음” + “공유로 저장해보세요” 가이드
    - Loading: skeleton grid
    - Error: 동기화 실패 배너 + 재시도

---

### S2. 저장(공유로 들어오는 핵심 화면)

- 목적: “저장 + 분류”를 한 번에 끝내기
- 진입:
    - 외부 공유 → 앱 열림 → 이 화면으로 바로
- UI:
    - 프리뷰 영역:
        - 대표 이미지(가능하면 첫 이미지)
        - 텍스트 미리보기(최대 3줄)
        - 출처 도메인/URL 표시
    - 분류:
        - 폴더 선택(최근/즐겨찾기 폴더 상단 고정)
        - 태그 입력(자동완성 + 최근 태그)
    - 메모(옵션): “한 줄 메모”
    - 버튼: [저장] [취소]
- 액션:
    - 저장 → 업로드/로컬 저장 → S1로 복귀 + 토스트
- 상태:
    - Empty preview: 공유 데이터가 빈 경우 → “텍스트 또는 URL을 붙여넣어 저장” fallback
    - Loading: 이미지 업로드 진행률
    - Error: 업로드 실패 시 “로컬에 임시 저장 후 재시도” 선택지

---

### S3. 폴더(보드) 관리

- 목적: 폴더 생성/이름변경/삭제
- UI:
    - 폴더 리스트(커버 이미지 1장 + 이름 + 개수)
    - [+ 새 폴더]
- 액션:
    - 폴더 탭 → 해당 폴더 필터로 S1 이동
    - 폴더 편집(이름 변경/삭제)
- 상태:
    - Empty: 폴더 없음 → “기본 폴더 만들기” 유도

---

### S4. 클립 상세

- 목적: “복기”에 최적화(명료하게)
- UI:
    - 상단: 대표 이미지 크게(스크롤)
    - 본문: 텍스트 전체, 메모, 태그 chips, 폴더 표시
    - 출처: 도메인 + [원문 열기]
    - 하단 액션: [태그 편집] [폴더 이동] [삭제]
- 상태:
    - Error: 원문 열기 실패(브라우저 문제) 안내

---

### S5. 태그 관리(선택)

- 목적: 태그 정리(중복/오타) — MVP에서는 “최근 태그” 정도만 있어도 됨
- UI:
    - 태그 리스트 + 검색
    - 태그 이름 변경/병합(병합은 Later로 빼도 됨)

---

### S6. 검색/필터

- 목적: “필요한 것만 다시 찾기”
- UI:
    - 검색 입력
    - 필터: 폴더 / 태그 chips
    - 결과: S1과 동일한 카드 그리드
- 검색 범위(MVP):
    - 텍스트, 메모, 도메인, 태그, (가능하면) 페이지 제목

---
