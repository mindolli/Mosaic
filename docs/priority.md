### Phase 0

1. MVP에서 “Tessera당 Mosaic 1개”로 갈지(단순) vs 다중 Mosaic(복잡)
2. 공유에서 들어오는 데이터 범위: URL/텍스트/이미지 중 “최소 보장”은 무엇인지
3. 인증: 게스트 모드 허용 여부(추천: 허용)
4. 개인 사용 용으로 개발 -> 서비스하는데 필요한 기능은 나중에 개발(일단은 인증 없이 게스트로)

---

### Phase 1. 기반(데이터/동기화 뼈대)

1. DB 스키마(Tessera/Mosaic/Tag/ImageAsset) 생성
2. 클라우드 스토리지 업로드/다운로드 파이프라인
3. 로컬 DB + SyncQueue 구조 잡기
4. 로그인(또는 게스트) + user_id 기준 데이터 분리

**완료 기준:** 로그인 후 빈 피드가 떠도 에러 없이 “내 공간”이 만들어짐

---

### Phase 2. “저장” 플로우 먼저(가치 경험 최우선)

1. Share → S2 저장 화면 진입(프리뷰 표시)
2. 저장 시: 이미지 업로드 + Tessera 생성 + 태그/Mosaic 연결
3. 저장 완료 후 S1에 카드가 보이게

**완료 기준:** 웹에서 공유 → 앱에서 저장 → 홈에 바로 보임

---

### Phase 3. “보기/복기” 플로우

1. S1 홈 피드(그리드/Masonry)
2. S4 상세 화면(원문 열기, 태그 편집, 삭제)
3. S3 Mosaic 관리(생성/삭제/이름변경)

---

### Phase 4. 다시 찾기(명료함 완성)

1. S6 검색(텍스트/도메인/태그) + 필터
2. 태그 자동완성/최근 태그 UX 개선

---

### Phase 5. 안정화/폴리시

1. 오프라인: 업로드 실패 시 로컬 임시 저장 + 재시도
2. 에러/로딩/빈 상태 UX 정리
3. (선택) URL 메타데이터 추출(제목/OG 이미지) 자동화
